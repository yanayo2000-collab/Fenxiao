# Fenxiao 运营使用手册

## 1. 适用范围
这份手册面向日常运营 / 风控 / 客服排查人员，用于说明当前 Fenxiao Web Console 可以做什么、怎么做、什么情况下不要乱点。

---

## 2. 当前后台可做的事

### 2.1 登录与查看
- 后台 session 登录
- 运营概览查看
- 奖励列表分页查询
- 风险事件分页查询
- 关系链查询
- 最近处理记录 / 审计列表查看

### 2.2 风险事件处理
当前支持 4 个动作：
- `处理`
- `忽略`
- `冻结用户`
- `解冻用户`

所有动作都会写入审计日志，并带上：
- 目标风险事件
- 动作名
- 处理备注
- 操作时间

---

## 3. 页面说明

### 3.1 奖励查询
适合查：
- 某个用户有没有奖励
- 某段时间奖励状态是否异常
- 风控后奖励有没有进入 `RISK_HOLD`

建议筛选项：
- 受益用户 ID
- 奖励状态
- 开始时间 / 结束时间
- 页码 / 每页条数

### 3.2 风险事件运营页
适合查：
- 某用户是否被打到风险事件
- 风险事件是待处理还是已忽略/已处理
- 风险动作是否已经执行

建议筛选项：
- 用户 ID
- 风险状态
- 开始时间 / 结束时间
- 页码 / 每页条数

### 3.3 关系链查询
适合查：
- 用户上级关系是否正确
- 用户当前关系是否被锁定
- 风险动作后关系状态是否发生变化
- 是否需要人工修正一级上级关系

如果确认关系绑定错了，当前后台已经支持：
- 人工修正一级上级用户 ID
- 留空后保存，将该用户设为根关系
- 自动写 relation 模块审计日志

注意：
- 已锁定关系不能手工修正
- 不要把用户改到自己的下游链路里，避免形成循环关系
- 修正前建议先写清备注，方便后续复核

### 3.4 最近处理记录
适合查：
- 最近是谁处理了什么风险
- 有没有误操作
- 某个动作是否真的落库

---

## 4. 风险动作说明

### 4.1 处理
适用于：
- 该风险已经确认，需要留档关闭
- 不需要进一步冻结/解冻，仅做处理完成标记

注意：
- 只允许对 `PENDING` 风险事件执行

### 4.2 忽略
适用于：
- 明确判断是误报 / 假阳性
- 该风险事件不应再触发运营动作

注意：
- 只允许对 `PENDING` 风险事件执行
- 被忽略的风险事件，不能再拿来冻结用户

### 4.3 冻结用户
适用于：
- 已确认该风险需要进入风控处理
- 需要让相关奖励进入风险挂起状态

执行后会联动：
- 用户状态转为风险态
- 风险事件进入已处理
- 相关奖励进入 `RISK_HOLD`
- 最近处理记录新增一条审计

注意：
- 只允许对 `PENDING` 风险事件执行
- 不要把“待观察”直接当成冻结依据
- 冻结前建议填写清楚处理备注

### 4.4 解冻用户
适用于：
- 已确认之前冻结为误判
- 或人工复核后决定恢复正常状态

执行后会联动：
- 用户恢复正常态
- 风险挂起奖励恢复为 `FROZEN` 或 `AVAILABLE`
- 最近处理记录新增一条审计

注意：
- 只有在系统识别到该用户确实存在风险挂起奖励时才允许解冻
- 不要对未冻结用户反复点击解冻

## 5. 推荐排查路径

### 场景 A：用户说奖励不见了
1. 先查奖励列表
2. 看状态是否为 `RISK_HOLD`
3. 再查风险事件
4. 如果命中风险事件，再决定是处理、忽略还是解冻
5. 最后看最近处理记录，确认动作是否落地

### 场景 B：用户说关系链异常
1. 查关系链
2. 看当前锁定状态
3. 如果伴随风险事件，再查风险列表
4. 必要时结合最近处理记录确认是否有人做过冻结/解冻
5. 如果确认一级上级绑错，且关系仍未锁定，再做人工修正并补备注

### 场景 C：复核一个风险事件
1. 先看风险详情和时间
2. 再看奖励状态是否已被联动影响
3. 决定是：处理 / 忽略 / 冻结 / 解冻
4. 填写简短但明确的备注

---

## 6. 处理备注建议
建议备注尽量短，但要包含判断依据。推荐格式：
- `人工复核通过，误报忽略`
- `确认异常收益，执行冻结`
- `用户申诉通过，恢复正常`
- `仅留档，无需进一步动作`

不建议只写：
- `已处理`
- `看过了`
- `ok`

---

## 7. 当前边界
当前这版已经是运营可用 MVP，但仍然有边界：
- 还没有更细的后台权限模型
- 还没有动作审批流
- 还没有 Linky 正式收益同步接入
- 还没有更复杂的风控规则编排

所以当前使用原则是：
> 先解决日常排查与基础处置，不把后台流程做过重。

---

## 8. 上线前最小检查清单
- 后台能正常登录
- 奖励查询可返回数据
- 风险查询可返回数据
- 风险动作能执行
- 最近处理记录能看到新增动作
- 奖励/关系链会在动作后联动刷新
- 前端 build 通过
- 后端测试通过
